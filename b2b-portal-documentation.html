Общая архитектура

B2B-портал для партнёров компании IPMATIKA Bel. Позволяет партнёрам создавать заявки на проекты, общаться с менеджерами в чате, отслеживать статусы. Менеджеры управляют проектами, оборудованием, рассылками. Администраторы управляют пользователями, настройками, БД и бэкапами.

Frontend

  • React 19 + Vite 7 + TypeScript 5.9
  • Tailwind CSS 4 + Roboto
  • TanStack Query v5 (серверный кэш)
  • Zustand v5 (локальный стейт: auth, chat)
  • ky (HTTP-клиент с поддержкой httpOnly cookies)
  • Socket.IO client (чат, онлайн)
  • Zod v4 (валидация)

Backend

  • Express 5 + TypeScript 5.9
  • Prisma 7 ORM
  • PostgreSQL 18 (Docker)
  • Socket.IO 4 (чат, events, онлайн)
  • JWT (access 15m) + Refresh Rotation (7d)
  • 2FA SMS (A1 Smart Sender)
  • Helmet CSP, rate limiting
  • Winston + Daily Rotate File (логи)

Socket.IO — Архитектура

Комнаты

  • user_{id} — личные события (block, superseded, status)
  • admin_room — ADMIN + MANAGER (статистика, проекты, статусы)
  • project_{id} — участники чата (сообщения)

События сервер → клиент

  • stats_updated — статистика онлайн (admin_room)
  • user:online / user:offline — статусы пользователей (admin_room)
  • user_blocked / user_unblocked_by_admin — блокировка (user_{id})
  • session_superseded — сессия вытеснена (user_{id})
  • project_created / project_updated / project_status_changed — изменения проектов
  • new_message — новое сообщение в чате
  • messages_read — прочтение сообщений

Data Flow — все сценарии

Каждый сценарий описан как цепочка: Client → Route [Middleware] → Controller → Service → DB → [Socket] → Response

1. Аутентификация

1.1 Регистрация пользователя (ADMIN)

POST/api/auth/registerADMIN only
Client → AdminCreateUser → POST с данными пользователя
authMiddleware → adminMiddleware → validate(registerSchema)
authService → registerUser(): проверка уникальности email/УНП/телефона
Prisma → bcrypt.hash(password), user.create()
Email → sendWelcomeEmailToUser (nodemailer)
Response → 201 { user }

1.2 Логин

LoginPage → POST /api/auth/login (email + password)
authService → loginUser(): findUnique → проверка блокировок → bcrypt.compare
Prisma → failedLoginAttempts++ → при 5+ lockUntil на 15 мин
USER → { requires2FA: true, userId }
ADMIN/MGR → generateTokens(): JWT 15m + refresh UUID 7d → httpOnly cookies
Socket → session_superseded → user_{id}, user:online → admin_room, stats_updated

1.3 2FA — отправка кода

LoginPage → POST /api/auth/2fa/send (rate: 3/min)
send2FACodeService → проверка 2FA lock, rate-limit 60s
Server → generate 6-digit code → bcrypt.hash(code)
A1 SMS → Smart Sender: IPMATIKA: код подтверждения XXXXXX
Response → { debugCode } (всегда, для отладки)

1.4 2FA — верификация

LoginPage → POST /api/auth/2fa/verify (code)
verify2FACodeService → bcrypt.compare(code, hash) → 3 attempts → lock 15 min
✅ Success → generateTokens() → httpOnly cookies
Prisma → twoFactorVerified: true, сброс попыток/хеша
Socket → session_superseded, user:blocked_status_changed, stats_updated, user_status_changed

1.5 Logout

Client → POST /api/auth/logout
logoutUser() → revokeUserRefreshTokens() → отзыв ВСЕХ refresh токенов
Prisma → user.update(): lastSeen = now - 10min, currentSessionId: null
Socket → user_status_changed → admin_room, stats_updated
Response → 200, чистит jwt cookie

1.6 Забыли пароль / Сброс пароля

LoginPage → POST /api/auth/forgot-password (email)
forgotPasswordService() → uuidv4 resetToken + 1 час expiry → email со ссылкой
Email link → POST /api/auth/reset-password (token + newPassword)
resetPasswordService() → findFirst по токену → bcrypt.hash → update

1.7 Смена пароля

ChangePasswordPage → POST /api/auth/change-password [auth]
changePasswordService() → bcrypt.compare(current) → bcrypt.hash(new) → update

1.8 Refresh Token Rotation

ky interceptor → 401 → POST /api/auth/refresh (cookie)
rotateRefreshToken() → SHA-256 → findUnique → проверка revokedAt
⚠️ Reuse → revokedAt !== null → отзыв ВСЕХ токенов пользователя
✅ Valid → prisma.$transaction([отозвать старый, создать новый])
Response → новый access JWT + refresh cookie

1.9 Профиль

Client → GET /api/auth/profile [auth] → req.user (без password)

1.10 Валидация и ошибки

Email-валидация: z.string().regex(/@/) вместо z.string().email() — Zod .email() требует TLD ≥2 символов, d@e.t не проходит.
Field-level ошибки: в catch-блоках проверять errorBody?.errors?.length > 0 и показывать errors[0].message вместо errorBody.message.
Сетевые ошибки: хелпер getErrorMessage(err, fallback) из UIHelpers.tsx. TypeError → «Сервер недоступен», TimeoutError → «Сервер не отвечает», HTTPError → fallback.
Исправлено 11 catch-блоков в 7 файлах. err.message || fallback больше не используется.

2. Управление пользователями (ADMIN)

2.1 Список пользователей

2.2 Удаление пользователя

deleteUserById() → проверка id !== self → prisma.user.delete()
Socket → stats_updated → admin_room

2.3 Блокировка/разблокировка

Admin → PATCH /api/user/users/:id/block [auth, admin]
toggleBlockUser() → 3 режима в одном endpoint'е:
🔓 lockUntil > now → снять брутфорс-блокировку (lockUntil: null, failedLoginAttempts: 0)
🔒 Иначе → ручная блокировка: isBlocked = !isBlocked, currentSessionId: null
Socket → user:blocked_status_changed, user_blocked / user_unblocked_by_admin, stats_updated

2.4 Смена пароля (force change)

ForcePasswordChange → POST /api/user/change-password [auth]
changeUserPassword() → bcrypt.hash → update (mustChangePassword: false)

JWT Refresh в Socket.IO middleware: при истечении access-токена (15 мин) middleware автоматически выполняет refresh через refreshToken-куку. SHA-256 верификация, ротация токенов, установка нового JWT через Set-Cookie. Без этого фикса socket-соединение падало каждые 15 минут и пользователи пропадали из списка онлайн.
